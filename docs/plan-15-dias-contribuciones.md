## Commit #3: Repos Nuevos en Preparacion

Ademas de los 2 commits diarios a repos existentes, se agrega un 3er commit diario
para **repos nuevos que se van preparando localmente** y se publican progresivamente.

Flujo:
1. Se prepara el repo localmente en `prweb/public/` (scaffold, codigo, sanitize)
2. Se hace el primer commit local
3. Se publica en GitHub cuando este listo (1 push = 1 commit nuevo en el graph)
4. Los dias siguientes se le agregan mejoras (README, tests, examples)

Esto permite ir construyendo repos nuevos sin prisa, y cada push cuenta como actividad.

Los repos nuevos candidatos se deciden en conversacion. El plan de 2 commits/dia
para repos existentes NO se interrumpe - el 3er commit es adicional.

**Meta ajustada:** 3 commits/dia = 45 commits en 15 dias.

## Notas

- Topics se agregan via `gh repo edit REPO --add-topic TOPIC` (no requiere clone)
- Repos nuevos (whatsapp, video) reciben mas atencion los primeros dias
- Priorizar contribuciones que mejoren descubrimiento (topics, README, badges)
- Cada commit debe ser real y util, no relleno
- Si un dia hay algo mas urgente, se puede mover al siguiente pero no saltar
- El 3er commit diario (repo nuevo) se coordina en conversacion
