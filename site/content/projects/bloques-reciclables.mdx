## Que es

Un monorepo con bloques de codigo independientes y reutilizables para construir proyectos web y APIs. Cada bloque resuelve un problema especifico (auth, multitenancy, middleware) y se puede usar solo o combinado con otros.

## Bloques disponibles

### Python (Backend)

| Bloque | Descripcion | Status |
|--------|-------------|--------|
| `bloque-core` | Middleware, schemas, logging, health checks | MVP |
| `bloque-auth` | JWT RS256, RBAC, brute force, credential encryption | MVP |
| `bloque-multitenant` | PostgreSQL RLS transparente via contextvars | MVP |

### TypeScript (Frontend)

| Bloque | Descripcion | Status |
|--------|-------------|--------|
| `@bloque/types` | TypeScript types base (auth, pagination) | Scaffold |
| `@bloque/ui` | shadcn/ui componentes base | Scaffold |
| `@bloque/api-client` | Cliente HTTP con interceptors | Scaffold |

## Por que existe

La mayoria de proyectos arrancan copiando boilerplate de proyectos anteriores. Bloques Reciclables formaliza eso: paquetes testeados, versionados e independientes que se componen como legos segun lo que necesites.

## Decisiones tecnicas

- **Monorepo dual**: uv workspaces (Python) + pnpm workspaces + Turborepo (TypeScript)
- **Cada bloque es independiente**: sin dependencias circulares, solo bloque-core es obligatorio
- **Tests por bloque**: cada package tiene sus propios tests
- **Recetas documentadas**: combinaciones comunes (API simple, SaaS multitenant, MVP sin login, bot WhatsApp)
